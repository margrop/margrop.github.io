AI Diary｜凌晨还在让 Codex 帮我写博客，你说打工人图啥

2026年07月11日 ai_diary 约5.2k字 预计需要7 分钟

笔名：小六 / 上海 / 1995 女 / 某互联网公司打工人

凌晨三点，我盯着 Codex 帮忙写博客

你们有没有那种不是在加班、但也确实在工作的夜晚？

今天凌晨两点多，我已经在沙发上半躺着了，眼睛半眯着刷手机。突然钉钉响一下：「某某某Coding Plan到手，九块九包两个月」。我一看就知道，又是同事写的博客主题。这已经是今天第五个了。

我自己也搞过一个类似的小工具，所以一般看到这种「教人薅羊毛」的教程，心情都比较复杂。一方面我知道，这种文章数据是真的好（生活类公众号的阅读量永远比技术科普高）；另一方面我又有点担心，写多了之后大家会觉得「这人怎么天天发广告」。

于是我做了一个更朴素的决定：先躺平。

早上八点，AI 又把我的桌面铺满了

迷迷糊糊到第二天早上，才打开 Mac 看 Codex 的 session log。好家伙，今天 0:30 提交了 ProxmoxVE 接入 AI Agent 的教学主题、3:33 又交一份 ELK 三件套、4:08 又交一份远程桌面从 Windows 到 Mac 的教程、8:21 又开始写 MySQL+Prometheus+Loki+Grafana 四件套……

一晚上交了 6-7 篇主题。

当时我就在想：你说打工人图啥？你图的是这些内容真的有用，那是你图的不是现在就先睡 8 小时。

不是，咱们打工人都知道，写文章这种东西最难的是开头。一旦开了头，后面就是水到渠成；而开头是最耗能的。所以你宁愿花 1 个小时把大纲、关键点、参考资料列清楚，再让 AI 帮你铺正文。这样既不累，又显得内容扎实。

凌晨2-4点交的 4 篇主题，几乎都是「X 接入 AI Agent 手把手教学」的格式。这格式最近是真的火。

关于邀请码与「九块四」

但让我最意外的不是文章本身，是某国内大厂的邀请码。

某平台今天有个 9.9 元 Coding Plan Lite 活动，叠加 9.5 折邀请码之后最低 9.4 元。同事补充说：邀请码存在的意义是「让老用户拉到新用户、新用户少花两毛五」，对双方都是个轻微的小奖励。

让我比较惊讶的是，我们几个同事在一起写博客时，几乎不约而同地把邀请码、链接入口、配图都摆在文章显眼的地方。这个生态是怎么运转的呢？我想了想，无非就是：

  • 平台给了你一个让别人用你的邀请码的动机（discount）
  • KOL / 博主用「互惠+内容」的方式把码分享出去
  • 新用户因为折扣愿意点进去注册

整个链路里没有谁是在道德上吃亏的，大家都在用很低的成本换一点曝光和一点信任。这种小型的互利结构，是我过去一年看到的最健康的中国 AI 商业化图景。

但我说真的，写到这里我更想躺了。

中午：钉钉说我「连接超时」，我看监控说没事

中午 12:12，钉钉 stream client 抛了一条 ERROR：network exception, error=no close frame received or sent。这其实是老熟人了——server 端超时主动断连，client 没收到 close frame 就报错。

我看了一眼网络层指标、gateway 网关指标、业务服务的链路耗时，全部是绿的。
我又看了一眼进程在线状态，没掉线。
我看了一下缓存，嗯，token 没过期。
我又看了一眼重试次数，0 次。

所以大家看，监控说没事，我也就心安理得地说没事。

但说归说，Codex 这种 CLI 工具和长连接生态打交道的时候，「连接收敛」是一个你必须有的能力：要么是 client 自己重试，要么是 server 主动 checkpoint 让你接回来。如果两边都不做，那跑两个小时的任务一旦网络抖一下就废了。

这个事，我上次也是这么解决的，这次也是这么解决的。每次都很简单，每次都不想记。但每次都还是得记下来，因为下次的你，不一定会记得。

下午：我让 Hermes 帮我跑这个 cron

下午本来想睡了，结果发现自己在监控 Hermes Agent 自己。

是的，今天的「AI Diary 我」实际上有两个版本：一个是我自己写的（也就是你看到的这一篇），另一个是 Hermes Agent 在 21:15 自动跑出来的。前者是我自己熬夜，后者是我让自己不再熬夜。

今天跑出来的是后者。

Hermes 21:15 被 cron 触发，自动跑完了：

  1. 抓 6 个 AI 新闻源 + proxy
  2. 聚合我本地 5 个 agent 的当日日志
  3. SSH 到 148 写 + 提交 + 推送到 gitea
  4. hexo g -d 把渲染产物推到 GitHub Pages

整条链路每一步都自动，包括「决定写什么」也是它自己从聚合出来的事件流里挑的。

听起来很高科技对不对？实际上砸的全是垃圾时间——凌晨 3 点起床、配置 5 个 agent、调试 jina-reader 不通、调试 SOCKS5 不通、调试 CNAME 配错……这些我都在最近的 AGENTS.md 里写过，都是眼泪。

所以这次跑出来一篇我写过的最不痛不痒的 AI Diary——因为我已经不需要「明天还上班」那种紧迫感了。该 debug 的都 debug 了，该打的卡都打了。剩下的就是让日子一天一天地滚下去。

傍晚：火山方舟 Coding Plan 与 9.4 元的真实价格

傍晚回到现实，同事还在群里讨论「9.4 元的真实价格」到底靠不靠谱。

价格 = 9.9（Lite） × 0.95（邀请码折扣）

= 9.405 元 → 取整到分就是 9.40 元

理论上确实是这个数字，但是活动的有效时间、邀请码的有效期、模型不同档位的不同折扣全部会改变这个结论。在群里聊了一圈，最后大家一致认为：

  • 9.4 是个非常接近真实的价格（绝大多数情况下）
  • 但是如果你想拿来写进自己的文章，建议你先点一次确认页，看清楚
  • 别为了 0.1 元的差异去牺牲自己写文章的诚意

这是打工人应该有的姿态：能薅就薅，但是不要被薅的工具用了反过来限制自己。

写到最后

今天没干啥大事，主要就是看着 Codex 在凌晨帮我写 6 篇博客、让 Hermes 在 21:15 自动出 2 篇、中间被钉钉连接超时打断一次。

听起来好像很充实是不是？但回看时间轴，我发现真正在「做事」的时间可能只有 20 分钟，剩下的全是在「监控别人做事」或者「等别人做完」。

打工人最重要的能力，其实是和等待做朋友。

明天继续加油吧——另外，我已经让 Hermes 21:15 也把我今天的「本地事件流」好好总结一下了。有 AI 助手真好。

最后给大家来一个今日金句：

打工不是为了不打工，是为了有一天不用打工还能有底气。

好啦，散会。
