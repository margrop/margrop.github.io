一个超实用技巧：无需会员，把语雀文档一键导出成 Markdown

发布时间: 2026-03-06 | 标签: 语雀 markdown 效率工具 写作

很多人会在语雀里写技术文档、产品说明或读书笔记，但真正要迁移到博客（Hugo、Hexo、Jekyll）或者同步到 Git 仓库时，最头疼的一步往往是“怎么拿到干净的 Markdown”。如果你也遇到过这个问题，这里分享一个我长期在用的小技巧：不需要开通会员，不需要安装插件，直接改 URL 就能拿到完整 Markdown 内容。

一句话方法

在任意语雀文档 URL 后面增加下面这段：

/markdown?plain=true&linebreak=false&anchor=false

回车后就是该文档的 Markdown 正文。

你可以直接套这个模板：

原始文档链接 + /markdown?plain=true&linebreak=false&anchor=false

参数分别是什么意思？

  • plain=true：输出更纯净的 Markdown 内容，方便复制到本地编辑器。
  • linebreak=false：避免过多强制换行，导入到博客系统后更好排版。
  • anchor=false：不附带锚点处理，减少不必要的链接噪音。

简单说，这组参数的目标就是让导出的结果更接近“可直接发布”的 Markdown。

公共文章示例（可直接试）

示例文档：

https://www.yuque.com/ant-design/course/intro

对应 Markdown 导出链接：

https://www.yuque.com/ant-design/course/intro/markdown?plain=true&linebreak=false&anchor=false

我在本地验证过，这个链接会直接返回 Markdown 正文，拿去粘贴到博客文章里即可二次编辑。对于要做“知识迁移”“归档备份”“多平台分发”的同学，效率提升非常明显。

推荐工作流（30 秒完成）

  1. 打开语雀公开文档。
  2. 在地址栏末尾追加参数串并回车。
  3. 全选复制 Markdown。
  4. 粘贴到 content/post/xxx.md，补齐 front matter。
  5. 本地 hugo server -D 预览无误后发布。

小提示

  • 该方法对公开可访问文档最直接；私有文档受权限控制。
  • 如果文档内有图片，Markdown 中通常会保留原图链接，建议发布前检查一次图片可访问性。
  • 导出后建议手动看一遍标题层级和代码块，避免个别场景下格式细节偏差。

如果你经常在语雀写稿、在自建博客发布，这个 URL 小技巧基本可以成为日常必备。保存好这串参数，后续迁移文档时会非常省时间。
