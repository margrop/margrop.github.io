Published: 2021-03-27
java springboot undertow

The problem occurs

Error reason

In a Linux system, when the Spring Boot application is started with the java -jar command, a tomcat (or undertow) will be generated in the /tmp directory of the operating system. The uploaded file must first be converted into a temporary file and saved in this folder. Because the files in the temporary /tmp directory are not used for a long time (10 days), they will be automatically deleted by the system mechanism. Therefore, if the system has not used the temporary folder for a long time, it may cause the above problem.

Solution

Method 1: Temporary solution

Restart the SpringBoot application to solve the problem immediately.

Method 2: Modify application.yml

Add the configuration spring.servlet.multipart.location to the configuration file applicaiton.yml and set it to a fixed upload file directory.

spring:
  servlet:
    multipart:
      location: /data/app
